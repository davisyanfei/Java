Description

For this Homework, you will be implementing a single class: MazeNavigator. This class will process a file input to navigate a maze for users. 

Note: 5 points of your Challenge grade is based on Coding Style.  You will need to follow the standards described on Brightspace . Use the "Run" button to check your Coding Style without using a submission. 

Instructions

This program will navigate a character through a 10 x 10 maze! The characters moves will be determined by 1 or more players, whose move selections will be stored in a file available to the program. The character will be represented by an 'X', which will move through the blank spaces in the maze as moves are processed. Here is an example:

In this example, there were two moves. Player 2 moved left and Player 1 moved right. The final position of the character by the end of the run is the same. As the solution is concurrent, the order of players is not pre-determined. In the next run, Player 1 may go first. The moves for each player are included in their own files. Your solution will synchronize access to the maze to ensure the player threads do not encounter race conditions. 

Requirements Overview: 

  • Create a class that utilizes static variables in a deterministic and multithreaded manner. Your solution must have five static variables: 
    • currentRow: An int, the current row associated with the maze position. The initial position is 4. 
    • currentColumn: An int, the current column associated with the maze position. The initial position is 4.
    • moveNumber: An int, the number of moves made so far. 
    • started: A boolean, the status of whether the maze navigation has started yet. 
    • maze: A char[][], the 10 x 10 char array that represents the maze. 
  • The class must extend Thread. 
  • Objects of the class will be passed an integer player number and a String filename containing the moves for that player. 
  • Within the run() method, the class must iterate through each line of the file and apply the move to the character in the maze. 
  • The processing described above should happen concurrently for multiple threads. 
    • The only synchronized portions of the solution should be for accessing shared resources or printing to the console. 
    • Do not synchronize the entire run method. You will not receive credit if you do so. 

Class Requirements: 

MazeNavigator must extend Thread.

Implementation Details

The specifics of the implementation are up to you. The general requirements are below. 

  • Remember, all access to shared resources must be synchronized to prevent race conditions. 
  • Before the game begins, print the initial maze with a welcome message. 
    • To initialize the maze, set the element at [4][4] to 'X'. All other elements should be set to a space (' '). 
    • During printing, be sure to separate elements with commas and include opening and closing brackets for each line. For an example, review the sample output. 
    • Any thread may do this, but it should only be done once! 
  • Begin reading the file. Each line contains a move that corresponds to an integer between 1 and 4, inclusive. 
    • 1: Left
    • 2: Right
    • 3: Up
    • 4: Down
    • Any other value should be treated as an error. Print "Error, invalid input!" and do not modify the maze for that turn. 
  • If the move is valid, move the 'X' character to the new position. Set the previous element to a space (' '). 
  • Print the move number and player number. Additionally, print the updated maze, separating elements by a comma, with square braces surrounding each row []. 
  • That's it! Repeat for as many moves as are included in the file. 

Testing

No local test cases are provided with this assignment. However, we provide the following main method and sample test files as examples: 

Main Method

public static void main(String[] args) {
    try {
        MazeNavigator[] mazeNavigators = {new MazeNavigator(1, "PlayerOneMoves.txt"),
                new MazeNavigator(2, "PlayerTwoMoves.txt")}; 

        for (int i = 0; i < mazeNavigators.length; i++) {
            mazeNavigators[i].start();
        }
        for (int i = 0; i < mazeNavigators.length; i++) {
            mazeNavigators[i].join();
        }
    } catch (Exception ex) {
        ex.printStackTrace();
        return;
    }
}

Testing Recommendations

  • Create additional threads with new files.
  • Change the file contents, experiment with 1 or more lines in each. 
  • Verify that your solution is actually concurrent. It should not process each player in order or rotate between them in a pre-determined manner.

Notes: 

  • Files may contain any number of moves. Players may have different numbers of moves. 
  • If a move will leave the valid range of indexes in the array, loop back around. That is, incrementing 9 would result in 0, or decrementing 0 would result in 9.
  • If a move is not one of the valid options (1, 2, 3, or 4), print "Error, invalid input!" and do not modify the maze for that turn.   
    • Files will only contain integers. 
    • You do not need to include a main method in your submission, but you should use one during testing. 
    • You should follow the requirements documented in the Debugging handout for preventing race conditions. Solutions that use excessively broad scope are not considered concurrent and will not receive credit. 
    • Remember the difference between static and non-static. As there will be multiple MazeNavigator objects, you will need to find a way to implement concurrency among all of them. If your solution does not implement concurrency properly for multiple objects, it will not receive credit. 
    • You must extend Thread for this assignment.
