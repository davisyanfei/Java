Description

For this Homework, you will be implementing one class: Paint that will be a more advanced implementation of the two we covered in the walkthrough and debugging exercises. 

Note: 5 points of your Challenge grade is based on Coding Style.  You will need to follow the standards described here.  Use the "Run" button to check your Coding Style without using a submission. 

Instructions

All input and output should be done through the GUI. Any input and output conducted via the terminal will not receive points. 

Starter Code

Layout

Your program should create a GUI paint application with the following features:

  • The GUI appears in center of screen
  • Top Panel has 4 buttons: Clear, Fill, Erase, Random
  • Middle section is white canvas
  • Bottom Panel has Hex TextField, Hex Button, R TextField, G TextField, B TextField, RGB Button

Implementation Instructions

  • Use code from Walkthrough to achieve this.
  • The size of the pen stroke should be set to 5.

  • Set the pen color equal to the current background color.

  • Change the pen color to a randomly generated color.

Debugging

What might cause an Exception in thread "AWT-EventQueue-0" java.lang.NullPointerException?

  • You are attempting to access graphics2D inside an action listener
    • Solution: change graphics2D values in a separate method (see debug code color methods)
  • You are attempting to access text or button values inside a color changing method (ie. clear(), fill(), etc)
    • Solution: make changes to text or button values inside the action listener instead of a separate method
  • You are attempting to reference a method directly instead of using the paint object to call it
    • Solution: an instance of the class paint needs to call the methods - ex. paint.clear() not clear()

Random Numbers

We will be using the random number generator available in Java.util.Random;. Import it and declare a new Random object in the following format: Random random = new Random();. To retrieve a random integer from this object, you can use random.nextInt(). You'll need to figure out how to generate a number within a range of acceptable values (valid RGB).

Note: Random numbers generated this way are not truly random. Creating functionally random numbers is a significant issue in computer science and cryptography. You are welcome to make your numbers more random if you'd like to research ways to do so!
